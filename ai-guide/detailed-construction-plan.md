# 「窺天命」PWA 詳細施工計劃

## 階段 4：IChing JSON 資料建立 (詳細計劃)

### 4.1 建立 hexagrams.json (64卦資料)

**目標**：創建完整的64卦數據庫

**任務分解**：
1. 定義數據結構
2. 創建64卦基本資料
3. 包含卦辭、象辭、關鍵詞、摘要
4. 驗證數據完整性

**數據結構**：
```json
{
  "id": 1,
  "name": "乾為天",
  "upper": "乾",
  "lower": "乾",
  "binary": "111111",
  "judgment": "乾。元亨利貞。",
  "image": "天行健，君子以自強不息。",
  "keywords": ["剛健", "主動", "開創", "領導"],
  "summary": "乾卦代表純陽剛健，象徵天道運行，君子應效法天道，自強不息。"
}
```

**驗證標準**：
- 64卦數量正確
- 每卦包含所有必要字段
- binary格式正確 (6位二進制)
- 卦辭、象辭準確

### 4.2 建立 lines.json (384爻資料)

**目標**：創建完整的384爻數據庫

**任務分解**：
1. 定義數據結構
2. 創建每卦6爻資料
3. 包含爻辭、白話解釋、關鍵詞
4. 驗證數據完整性

**數據結構**：
```json
{
  "hexagramId": 1,
  "line": 1,
  "text": "初九，潛龍勿用。",
  "plain": "潛藏的龍，時機未到，不宜行動。",
  "keywords": ["潛藏", "等待", "準備"]
}
```

**驗證標準**：
- 384爻數量正確
- 每爻包含所有必要字段
- 爻辭準確
- 白話解釋通順

### 4.3 建立 hexagram-relations.json (卦象關係)

**目標**：定義每卦的互卦、錯卦、綜卦、配對關係

**任務分解**：
1. 計算每卦的互卦
2. 計算每卦的錯卦
3. 計算每卦的綜卦
4. 定義配對關係
5. 驗證關係正確性

**數據結構**：
```json
{
  "1": {
    "nuclear": 1,
    "opposite": 2,
    "reverse": 1,
    "pair": 2
  }
}
```

**驗證標準**：
- 每卦都有四種關係
- 關係計算正確
- 數據完整性檢查

### 4.4 建立 change-map.json (變卦路徑)

**目標**：定義每卦的64種變卦組合

**任務分解**：
1. 定義變卦邏輯
2. 計算每種動爻組合的變卦
3. 創建變卦映射表
4. 驗證變卦正確性

**數據結構**：
```json
{
  "1": {
    "none": 1,
    "1": 44,
    "2": 13,
    "3": 10,
    "4": 9,
    "5": 14,
    "6": 43,
    "1,2": 33,
    "1,2,3,4,5,6": 2
  }
}
```

**驗證標準**：
- 每卦有64種變卦
- 變卦邏輯正確
- 數據完整性檢查

### 4.5 建立 iching-templates.json (易經模板)

**目標**：創建易經解釋模板系統

**任務分解**：
1. 定義模板結構
2. 創建關鍵詞模板
3. 創建建議模板
4. 創建風險模板
5. 驗證模板邏輯

**數據結構**：
```json
{
  "keywords": {
    "開創": {
      "score": 8,
      "advice": "適合開始新計劃，主動出擊。",
      "risk": "避免過於冒進，注意時機。"
    }
  }
}
```

**驗證標準**：
- 模板結構正確
- 關鍵詞覆蓋完整
- 建議和風險邏輯合理
