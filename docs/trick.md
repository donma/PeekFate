# 命理計分優化 Tricks

## 核心方法：迭代式失敗分析

```
跑驗證 → 找失敗 → 分類失敗原因 → 改一個參數 → 跑驗證 → 重複
```

不是預設規則然後祈禱有效，是用失敗案例反推規則。每次只改一個變數，否則無法確認哪個改動有效。

---

## Trick 1：事件分類比十神計算更重要

凶事失敗的最大原因不是十神算錯，是事件被分到 `default`（用正向十神配對），導致凶事拿高分。

**必須覆蓋的凶事 pattern：**

| 類別 | 關鍵詞 |
|------|--------|
| 死亡 | 逝世 |
| 逮捕 | 逮捕、收押、被捕、入獄、判刑、定罪 |
| 制裁 | 制裁、封殺、糾紛 |
| 逃稅 | 逃稅、漏稅、罰款、被罰 |
| 禁賽 | 禁賽、禁令、處罰 |
| 債務 | 拋售、債務、危機、爆發、破產 |
| 抗檢 | 抗檢、風波 |
| 裁員 | 裁員、解約、解僱 |
| 醜聞 | 醜聞、出軌、劈腿、外遇、涉賭、欠債 |
| 受傷 | 燒傷、失智、確診、重病 |
| 政爭 | 政爭、反對、抗議 |
| 攻擊 | 槍擊 |
| 失敗 | 退賽、摔倒 |

## Trick 2：凶事需要大乘數

```
death:    ×3.5（死亡是最強凶事）
sanction: ×3.0
attack:   ×3.0
failure:  ×3.0
```

乘數不夠 → 凶事分數壓不下去 → 超過 52 分 → 失敗。

---

## Trick 3：吉事需要低閾值 + 備援機制

**閾值：≥45（不是 ≥48）**

原因：很多吉事（結婚、就職、就任）發生在「普通日」，不是大吉日。

**備援機制：** 當流日十神不是 target 時，檢查日柱/時柱有沒有正官/正印/食神/正財，有就加分。

```
backupScore = 0
日柱十神 in [正官,正印,食神,正財] → +6
時柱十神 in [正官,正印,食神,正財] → +5
月柱十神 in [正官,正印,食神,正財] → +2
年柱十神 in [正官,正印,食神,正財] → +2
clamp(-5, 16)
```

## Trick 4：吉凶方向完全不同

- **吉事**：target = 正官/正印/食神/正財，avoid = 七殺/劫財/傷官
- **凶事**：target = 七殺/劫財/傷官，avoid = 正印/比肩

凶事不是「吉事的反面」，是完全獨立的十神配對邏輯。

---

## Trick 5：伏吟/反吟/空亡對凶事加分

```
伏吟（流日柱=出生日柱）→ 凶事 +5，吉事 +3
反吟（流日柱沖出生日柱）→ 凶事 +4，吉事 -2
空亡 → 凶事 +2，吉事 -1
```

---

## Trick 6：天德/月德/天赦穩定加分

不論吉凶，天德/月德/天赦各 +3，最多 +9。不需要調整。

---

## Trick 7：分數公式

```
吉事：score = 50 + (eventMatch × mult × 0.7) + (backupScore × 1.5)
凶事：score = 50 - (eventMatch × mult)
中性：score = 50 + (eventMatch × 1.5)

+ 天德/月德/天赦（±9）
+ 地支關係（±5）
clamp(0, 100)
```
