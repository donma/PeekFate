# 驗證方法說明

## 驗證目標

評估計分引擎對已知歷史事件的分類能力：給定一位名人的出生資料與一個已知事件日期，計分引擎是否能產生符合事件性質的分數區間。

## 使用資料

### 名人資料

- 來源：`data/celebrities/celebrities.json`
- 規模：252 位華人名人
- 類別：政治人物、演員、歌手、運動員、企業家、科學家、文學家、導演等
- 欄位：姓名、出生日期、出生時間（選填）、性別（選填）

### 事件資料

- 來源：`data/celebrities/verification-events.json`
- 規模：231 筆已知事件
- 分佈：正向事件約 154 筆、負向事件約 66 筆、中性事件約 11 筆
- 事件類型：就職、金牌、結婚、上市、逝世、逮捕、封殺、醜聞、債務危機等

## 事件分類

系統根據事件描述自動分類為 22 種子類型：

| 子類型 | 目標十神 | 避免十神 | 方向 |
|--------|---------|---------|------|
| inauguration | 正官、七殺 | 傷官、劫財、比肩 | 正向 |
| gold_medal | 食神、傷官 | 正印、偏印、劫財 | 正向 |
| major_award | 食神、傷官、偏財 | 正印、偏印、劫財 | 正向 |
| ipo | 正財、偏財、七殺 | 比肩、劫財、正印 | 正向 |
| founding | 偏財、七殺、比肩 | 正印、劫財 | 正向 |
| marriage | 正財、正官 | 劫財、七殺、傷官 | 正向 |
| creative_release | 食神、傷官 | 正印、偏印 | 正向 |
| death | 七殺、劫財、傷官 | 正印、偏印、比肩 | 負向 |

完整定義見 `js/engines/scoring-engine.js` 中的 `EVENT_PROFILES`。

## 計分公式

```
正向事件：score = 50 + (eventMatch × mult × 0.7) + (backupScore × 1.5)
負向事件：score = 50 - (eventMatch × mult)
中性事件：score = 50 + (eventMatch × 1.5)

+ 天德/月德/天赦（±9）
+ 地支關係（±5）
clamp(0, 100)
```

其中 `eventMatch` 為四柱十神與事件目標十神的加權匹配分數。

## 通過條件

| 事件類型 | 通過條件 |
|----------|---------|
| 正向事件 | 分數 ≥ 45 |
| 負向事件 | 分數 ≤ 52 |
| 中性事件 | 分數 45–60 |

## 回測結果

| 指標 | 數值 |
|------|------|
| 總事件數 | 231 |
| 通過數 | 218 |
| 回測通過率 | 94% |
| 正向事件通過率 | 151/154 (98%) |
| 負向事件通過率 | 56/66 (85%) |
| 中性事件通過率 | 11/11 (100%) |

## 回測限制

1. **非泛化測試**：目前驗證使用的是已知事件資料集，尚未拆分 train / test / holdout dataset。回測通過率僅代表對現有資料集的表現，不應宣稱為泛化能力。

2. **人工標註偏差**：事件分類由人工規則驅動，部分事件的分類可能受主觀判斷影響。

3. **資料集偏差**：名人資料以華人為主，事件類型以重大公共事件為主，可能不代表一般民眾的日常事件。

4. **非科學驗證**：本系統基於傳統命術規則，屬於民俗文化範疇，不構成科學驗證。

5. **不代表未來預測能力**：歷史事件回測結果不可解讀為對未來事件的預測準確率。
